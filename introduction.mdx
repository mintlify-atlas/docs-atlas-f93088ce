Wraft is an open-source Document Lifecycle Management platform built with Elixir and Phoenix. It gives organisations complete control over their most important documents—from the first draft through collaborative editing, multi-stage approval workflows, e-signatures, and final distribution. Everything in Wraft is built on open formats (Markdown and JSON), so your content remains accessible and future-proof regardless of how your tooling evolves.

## Who Wraft Is For

Wraft is designed for teams that produce high volumes of structured business content: legal teams managing contracts and compliance letters, operations teams issuing official correspondence, HR departments creating offer letters and policy documents, and developers who want a programmable document engine behind their own products. Rather than forcing authors into a generic word processor, Wraft lets you encode your organisation's document standards into reusable templates and enforce them through automated pipelines.

## Core Concepts

Understanding a few building blocks will help you navigate the API and the UI:

- **Layout** — A Typst or LaTeX template file that controls the visual appearance of a rendered PDF. Layouts are reusable across many content types.
- **Theme** — A set of typography and colour settings applied to a layout at render time.
- **Content Type** — A schema that binds together a layout, a theme, and an approval flow. It declares the typed fields that authors must fill when creating a document.
- **Instance (Document)** — A concrete document created from a content type. Instances hold field values, move through workflow states, accumulate comments, and can be built into a PDF at any point.
- **Flow & States** — A flow is an ordered sequence of states (e.g. Draft → Legal Review → Approved → Signed). States can have assigned approvers and transition rules.
- **Pipeline** — An automated processing chain that can bulk-build documents, trigger external services, or chain multiple content type builds together.
- **Data Template** — A Markdown body template pre-filled with default content, attached to a content type to speed up authoring.

## Architecture Overview

Wraft's backend is written in **Elixir 1.19.2** on top of the **Phoenix** web framework. The stack is designed for reliability and extensibility:

| Component | Technology | Purpose |
|---|---|---|
| API Server | Elixir / Phoenix | REST API, real-time channels, background jobs (Oban) |
| Database | PostgreSQL 18 | Relational data, audit logs, version history |
| Object Storage | MinIO (S3-compatible) | Document assets, uploaded files, rendered PDFs |
| Search | Typesense 27.1 | Full-text search across documents and templates |
| PDF Rendering | Typst 0.14.2 / XeLaTeX | High-quality typeset PDF generation |
| Document Conversion | Pandoc 3.9.0.2 | DOCX import and format conversion |
| Image Processing | ImageMagick | Asset manipulation and preview generation |
| PDF Signing | Java 21 | Cryptographic digital signatures on PDF documents |
| Native Extensions | Rust NIFs | Performance-critical native operations |
| Frontend | React (Next.js) | Web application served separately |

The backend exposes a versioned REST API under `/api/v1` with JWT authentication via Guardian. OpenAPI documentation is auto-generated and available at `/spec/swaggerui` on any running instance.

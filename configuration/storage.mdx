Wraft uses [MinIO](https://min.io) â€” an S3-compatible object storage server â€” as its primary file store. Every file produced or consumed by the document pipeline is persisted to MinIO: generated PDFs, layout files, theme assets, template source files, and public slugs. The application communicates with MinIO through the `ex_aws` and `waffle` libraries, which means any S3-compatible service (AWS S3, DigitalOcean Spaces, Cloudflare R2, and others) can be used in production by pointing the environment variables at it.

## Local Development Setup

<Steps>
  <Step title="Install MinIO">
    Download the MinIO server binary for your operating system from the [MinIO documentation](https://min.io/docs/minio/linux/index.html) and place it on your `$PATH`.
  </Step>

  <Step title="Start the MinIO server">
    Run the following command from your terminal. The `--console-address` flag enables the browser-based MinIO Console on port `9001`.

    ```shell
    minio server ~/minio --console-address :9001
    ```

    MinIO will store all data under `~/minio`. Substitute a different path if you prefer a different data directory.
  </Step>

  <Step title="Configure environment variables">
    Add the following to your `.env.dev` file, then source it:

    ```shell .env.dev
    export MINIO_URL=127.0.0.1:9000
    export MINIO_HOST=127.0.0.1
    export MINIO_PORT=9000
    export MINIO_BUCKET=wraft
    export MINIO_ROOT_USER=minioadmin
    export MINIO_ROOT_PASSWORD=minioadmin
    ```

    ```shell
    source .env.dev
    ```
  </Step>

  <Step title="Create the bucket">
    The `mix setup` command calls `mix wraft.bucket` automatically, which creates the primary bucket (named by `MINIO_BUCKET`) and uploads default public files. You can also run it manually:

    ```shell
    mix wraft.bucket
    ```

    <Note>
    `mix setup` handles bucket creation, database migration, and seed data in one step. During a fresh local setup, running `mix setup` is sufficient â€” you do not need to create the bucket separately.
    </Note>
  </Step>
</Steps>

## MinIO Console

Once the server is running, you can access the MinIO web console at:

```
http://localhost:9001
```

Log in with your `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD` credentials. The console lets you browse buckets and objects, manage access policies, and monitor server metrics.

## Docker Compose Setup

When running Wraft with `docker compose up`, MinIO runs as a container alongside the backend. A dedicated `createbucket` init container automatically creates both the primary and backup buckets on first start.

### Add MinIO to /etc/hosts

The Docker backend container resolves MinIO as `minio` (the Docker Compose service name). You must add this entry to your host machine's `/etc/hosts` so that presigned URLs generated by the backend resolve correctly in your browser:

<Tabs>
  <Tab title="macOS / Linux">
    ```shell
    echo "127.0.0.1 minio" | sudo tee -a /etc/hosts
    ```
  </Tab>
  <Tab title="Windows">
    Run the following command in an elevated (Administrator) Command Prompt:

    ```cmd
    echo 127.0.0.1 minio >> C:\Windows\System32\drivers\etc\hosts
    ```
  </Tab>
</Tabs>

### Docker Compose MinIO service

The MinIO container exposes two ports â€” the S3 API on `9000` and the web console on `9001`. Data is persisted to a named Docker volume:

```yaml docker-compose.yml (MinIO section)
minio:
  image: quay.io/minio/minio
  ports:
    - 9000:9000
    - 9001:9001
  environment:
    MINIO_ROOT_USER: ${MINIO_ROOT_USER}
    MINIO_ROOT_PASSWORD: ${MINIO_ROOT_PASSWORD}
    MINIO_SERVER_URL: http://127.0.0.1:9000
  volumes:
    - miniodata:/data:rw
  command: minio server /data --console-address ":9001" --address :9000
```

<Note>
In the Docker Compose backend service, `MINIO_HOST` is set to `minio` (the service name) rather than `127.0.0.1`, so containers communicate over the internal Docker network.
</Note>

## Production Configuration

In production, point the MinIO environment variables at any S3-compatible service. No code changes are required â€” the `ex_aws` library honours the `MINIO_HOST`, `MINIO_PORT`, and credential variables for all S3 API calls.

<Tabs>
  <Tab title="AWS S3">
    ```shell
    export MINIO_URL=https://s3.amazonaws.com
    export MINIO_HOST=s3.amazonaws.com
    export MINIO_PORT=443
    export MINIO_BUCKET=your-wraft-bucket
    export MINIO_ROOT_USER=AKIAIOSFODNN7EXAMPLE
    export MINIO_ROOT_PASSWORD=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
    ```
  </Tab>
  <Tab title="DigitalOcean Spaces">
    ```shell
    export MINIO_URL=https://nyc3.digitaloceanspaces.com
    export MINIO_HOST=nyc3.digitaloceanspaces.com
    export MINIO_PORT=443
    export MINIO_BUCKET=your-wraft-space
    export MINIO_ROOT_USER=your-spaces-access-key
    export MINIO_ROOT_PASSWORD=your-spaces-secret-key
    ```
  </Tab>
  <Tab title="Self-hosted MinIO">
    ```shell
    export MINIO_URL=https://minio.yourdomain.com
    export MINIO_HOST=minio.yourdomain.com
    export MINIO_PORT=443
    export MINIO_BUCKET=wraft
    export MINIO_ROOT_USER=your-access-key
    export MINIO_ROOT_PASSWORD=your-secret-key
    ```
  </Tab>
</Tabs>

<Warning>
Make sure the production bucket exists and that the credentials have `s3:GetObject`, `s3:PutObject`, `s3:DeleteObject`, and `s3:ListBucket` permissions before starting the application.
</Warning>

## System Backup Bucket

Wraft supports an optional automated backup feature that writes PostgreSQL dumps and MinIO snapshots to a separate backup bucket. Configure it with the following variables:

| Variable | Default | Description |
|---|---|---|
| `SYSTEM_BACKUP_ENABLED` | `false` | Set to `true` to enable scheduled backups |
| `MINIO_BACKUP_BUCKET` | `wraft-backups` | Name of the bucket where backup archives are stored |
| `BACKUP_RETENTION_COUNT` | `7` | Maximum number of backup snapshots to keep |

```shell
export SYSTEM_BACKUP_ENABLED=true
export MINIO_BACKUP_BUCKET=wraft-backups
export BACKUP_RETENTION_COUNT=7
```

<Tip>
The backup bucket should be a separate bucket from the primary `MINIO_BUCKET`. In Docker Compose, the `createbucket` init container creates both buckets automatically when `MINIO_BACKUP_BUCKET` is set.
</Tip>

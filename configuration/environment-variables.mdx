Wraft is configured entirely through environment variables. Copy `.env.example` to `.env.dev`, fill in the values, then run `source .env.dev` before starting the server. In Docker Compose deployments the same variables are consumed directly from the environment. Secret keys must be freshly generated for every installation â€” never reuse the placeholder values from `.env.example`.

## Application

These variables control the core runtime behaviour of the Phoenix application.

| Variable | Required | Default | Description |
|---|---|---|---|
| `FRONTEND_URL` | âś… Yes | `http://localhost:3000` | Full URL of the Wraft frontend application |
| `BACKEND_URL` | âś… Yes | `http://localhost:4000` | Full URL of the Phoenix backend API |
| `SECRET_KEY_BASE` | âś… Yes | â€” | 64-byte random key used to sign/encrypt cookies and sessions |
| `GUARDIAN_KEY` | âś… Yes | â€” | Secret key used by Guardian to sign JWT authentication tokens |
| `CLOAK_KEY` | âś… Yes | â€” | Base-64 encoded 32-byte key used to encrypt sensitive database fields |
| `MIX_ENV` | âś… Yes | `dev` | Mix environment â€” `dev`, `test`, or `prod` |
| `SELF_HOSTED` | âś… Yes | `true` | Set to `true` to enable self-hosted mode and disable cloud billing checks |
| `PHX_SERVER` | âś… Yes | `true` | When set, Phoenix starts the HTTP server on boot |
| `RELEASE_NAME` | âś… Yes | `wraft_doc` | OTP release name used to identify the running node |
| `WRAFT_HOSTNAME` | âś… Yes | `example.com` | Domain used as the sender domain in outbound email headers |

<Note>
Generate each secret key with the appropriate command â€” never share or commit these values to version control.
</Note>

<CodeGroup>
```shell Generate SECRET_KEY_BASE
mix phx.gen.secret
```

```shell Generate GUARDIAN_KEY
mix guardian.gen.secret
```

```shell Generate CLOAK_KEY
openssl rand -base64 32
```
</CodeGroup>

| Variable | Required | Default | Description |


## Digital Signing

These variables configure the local keystore used to apply digital signatures to generated PDF documents. All three are optional â€” when omitted, Wraft falls back to the default keystore at `priv/signature/doc_signer.p12`.

| Variable | Required | Description |
|---|---|---|
| `SIGNING_LOCAL_FILE_PATH` | Optional | Relative path (under `priv/keystore/`) to the `.p12` certificate file used for PDF signing |
| `SIGNING_LOCAL_PASSPHRASE` | Optional | Passphrase for the `.p12` certificate file |
| `SIGNING_KEY_ALIAS` | Optional | Key alias within the `.p12` file to use for signing. Defaults to `"1"` |

## System Backups

When enabled, Wraft can automatically back up the PostgreSQL database and MinIO bucket to a dedicated backup bucket. This feature is disabled by default.

| Variable | Required | Default | Description |
|---|---|---|---|
| `SYSTEM_BACKUP_ENABLED` | Optional | `false` | Set to `true` to enable scheduled system backups |
| `MINIO_BACKUP_BUCKET` | Optional | `wraft-backups` | Name of the MinIO bucket where backup artifacts are stored |
| `BACKUP_RETENTION_COUNT` | Optional | `7` | Number of backup snapshots to retain before deleting old ones |
| `BACKUP_CLOAK_KEY_LABEL` | Optional | `v1` | Label identifying which Cloak key version to use for backup encryption metadata |
| `BACKUP_MIN_FREE_DISK_BYTES` | Optional | `2000000000` | Minimum free disk space (in bytes) required before a backup is attempted |
| `BACKUP_REMOTE_RESTORE_ENABLED` | Optional | `false` | Enable remote restore functionality. Off by default to prevent SSRF exposure |
| `BACKUP_REMOTE_ALLOWED_HOSTS` | Optional | â€” | Comma-separated list of hostnames permitted for remote restore operations |

<Warning>
Backup artifacts are stored unencrypted (plain `tar` + `pg_dump`). Keep the backup bucket private and restrict access to authorised staff only.
</Warning>

Wraft is configured entirely through environment variables. Copy `.env.example` to `.env.dev`, fill in the values, then run `source .env.dev` before starting the server. In Docker Compose deployments the same variables are consumed directly from the environment. Secret keys must be freshly generated for every installation вЂ” never reuse the placeholder values from `.env.example`.

## Application

These variables control the core runtime behaviour of the Phoenix application.

| Variable | Required | Default | Description |
|---|---|---|---|
| `FRONTEND_URL` | вњ… Yes | `http://localhost:3000` | Full URL of the Wraft frontend application |
| `BACKEND_URL` | вњ… Yes | `http://localhost:4000` | Full URL of the Phoenix backend API |
| `SECRET_KEY_BASE` | вњ… Yes | вЂ” | 64-byte random key used to sign/encrypt cookies and sessions |
| `GUARDIAN_KEY` | вњ… Yes | вЂ” | Secret key used by Guardian to sign JWT authentication tokens |
| `CLOAK_KEY` | вњ… Yes | вЂ” | Base-64 encoded 32-byte key used to encrypt sensitive database fields |
| `MIX_ENV` | вњ… Yes | `dev` | Mix environment вЂ” `dev`, `test`, or `prod` |
| `SELF_HOSTED` | вњ… Yes | `true` | Set to `true` to enable self-hosted mode and disable cloud billing checks |
| `PHX_SERVER` | вњ… Yes | `true` | When set, Phoenix starts the HTTP server on boot |
| `RELEASE_NAME` | вњ… Yes | `wraft_doc` | OTP release name used to identify the running node |
| `WRAFT_HOSTNAME` | вњ… Yes | `example.com` | Domain used as the sender domain in outbound email headers |

<Note>
Generate each secret key with the appropriate command вЂ” never share or commit these values to version control.
</Note>

<CodeGroup>
```shell Generate SECRET_KEY_BASE
mix phx.gen.secret
```

```shell Generate GUARDIAN_KEY
mix guardian.gen.secret
```

```shell Generate CLOAK_KEY
openssl rand -base64 32
```
</CodeGroup>

## Database

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | вњ… Yes (prod/dev) | Full Postgres connection URL. Format: `postgres://username:password@host:port/database` |
| `DEV_DB_USERNAME` | вљ пёЏ Docker only | Postgres username for the Docker Compose `db` service |
| `DEV_DB_PASSWORD` | вљ пёЏ Docker only | Postgres password for the Docker Compose `db` service |
| `DEV_DB_NAME` | вљ пёЏ Docker only | Database name for the Docker Compose `db` service |

<Note>
`DEV_DB_*` variables are only read when running `docker compose up`. The `DATABASE_URL` is constructed automatically inside the Docker Compose backend service using these three values. For bare-metal development, set `DATABASE_URL` directly.
</Note>

```shell Example DATABASE_URL
export DATABASE_URL=postgres://postgres:postgres@localhost:5432/wraft-dev-db
```

### Production database tuning

The following variables are read only when `MIX_ENV=prod` and allow tuning the Ecto connection pool:

| Variable | Default | Description |
|---|---|---|
| `POOL_SIZE` | `10` | Number of database connections in the Ecto pool |
| `POOL_TIMEOUT` | `15000` | Milliseconds to wait for a connection from the pool before timing out |
| `DB_TIMEOUT` | `15000` | Milliseconds to wait for a database query to complete |
| `ECTO_IPV6` | вЂ” | When set to any value, enables IPv6 socket options for the database connection |

## Storage (MinIO)

Wraft uses MinIO вЂ” an S3-compatible object storage server вЂ” to store all generated PDFs, theme assets, layout files, and template assets.

| Variable | Required | Default | Description |
|---|---|---|---|
| `MINIO_URL` | вњ… Yes | `127.0.0.1:9000` | Full URL (host + port) used by the Waffle file uploader as the asset host |
| `MINIO_HOST` | вњ… Yes | `127.0.0.1` | Hostname used by `ex_aws` to connect to MinIO |
| `MINIO_PORT` | вњ… Yes | `9000` | Port used by `ex_aws` to connect to MinIO |
| `MINIO_BUCKET` | вњ… Yes | `wraft` | Name of the primary storage bucket |
| `MINIO_ROOT_USER` | вњ… Yes | `minioadmin` | MinIO root access key (equivalent to AWS `access_key_id`) |
| `MINIO_ROOT_PASSWORD` | вњ… Yes | `minioadmin` | MinIO root secret key (equivalent to AWS `secret_access_key`) |
| `MINIO_SCHEMA` | Optional | `http` | URL scheme (`http` or `https`) used by `ex_aws` for S3 API calls. Set to `https` when connecting to TLS-enabled MinIO or S3-compatible services. |

```shell .env.dev вЂ“ MinIO example
export MINIO_URL=127.0.0.1:9000
export MINIO_HOST=127.0.0.1
export MINIO_PORT=9000
export MINIO_BUCKET=wraft
export MINIO_ROOT_USER=minioadmin
export MINIO_ROOT_PASSWORD=minioadmin
```

## Email

Wraft sends transactional emails via Resend (default) or an SMTP relay. See the [Email configuration guide](/configuration/email) for full details.

| Variable | Required | Description |
|---|---|---|
| `RESEND_API_KEY` | вњ… Yes (if using Resend) | API key from [resend.com](https://resend.com). When set, Resend is used as the mail adapter. |
| `SMTP_HOST` | вњ… Yes (if SMTP) | Hostname of the SMTP relay (e.g. `smtp.gmail.com`). When set, the SMTP adapter is used and overrides Resend. |
| `SMTP_PORT` | Optional | SMTP port. Defaults to `587` when not set |
| `SMTP_USERNAME` | вњ… Yes (if SMTP) | SMTP authentication username |
| `SMTP_PASSWORD` | вњ… Yes (if SMTP) | SMTP authentication password |
| `SMTP_SSL` | Optional | Set to `true` to enable SSL on the SMTP connection. Defaults to `false` |

## Digital Signing

These variables configure the local keystore used to apply digital signatures to generated PDF documents. All are optional вЂ” when omitted, Wraft falls back to the default keystore shipped with the application.

| Variable | Required | Description |
|---|---|---|
| `SIGNING_LOCAL_FILE_PATH` | Optional | Path to the `.p12` certificate file, relative to the application's `priv/` directory (e.g. `signature/my-cert.p12`). Defaults to `signature/doc_signer.p12`. |
| `SIGNING_LOCAL_PASSPHRASE` | Optional | Passphrase for the `.p12` certificate file |
| `SIGNING_KEY_ALIAS` | Optional | Key alias within the `.p12` file to use for signing. Defaults to `"1"` |
| `SIGNATURE_JAR_FILE` | Optional | Path to the PDF-signing JAR, relative to the application's `priv/` directory. Defaults to `signature/pdf-signer.jar`. |

## System Backups

When enabled, Wraft can automatically back up the PostgreSQL database and MinIO bucket to a dedicated backup bucket. This feature is disabled by default.

| Variable | Required | Default | Description |
|---|---|---|---|
| `SYSTEM_BACKUP_ENABLED` | Optional | `false` | Set to `true` to enable scheduled system backups |
| `MINIO_BACKUP_BUCKET` | Optional | `wraft-backups` | Name of the MinIO bucket where backup artifacts are stored |
| `BACKUP_RETENTION_COUNT` | Optional | `7` | Number of backup snapshots to retain before deleting old ones |
| `BACKUP_CLOAK_KEY_LABEL` | Optional | `v1` | Label identifying which Cloak key version to use for backup encryption metadata |
| `BACKUP_MIN_FREE_DISK_BYTES` | Optional | `2000000000` | Minimum free disk space (in bytes) required before a backup is attempted |
| `BACKUP_REMOTE_RESTORE_ENABLED` | Optional | `false` | Enable remote restore functionality. Off by default to prevent SSRF exposure |
| `BACKUP_REMOTE_ALLOWED_HOSTS` | Optional | вЂ” | Comma-separated list of hostnames permitted for remote restore operations |

<Warning>
Backup artifacts are stored unencrypted (plain `tar` + `pg_dump`). Keep the backup bucket private and restrict access to authorised staff only.
</Warning>
